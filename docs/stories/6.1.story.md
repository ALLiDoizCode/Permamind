### Architecture Context

**Monorepo Structure:**
[Source: package.json:6-8 - Root package.json workspaces]
[Source: architecture/source-tree.md:4-94 - Project directory structure]

Current monorepo configuration:
- Root workspace: Shared dev dependencies (TypeScript, ESLint, Prettier, Jest)
- CLI workspace: `cli/` with own package.json for CLI-specific deps
- Frontend workspace: NEW - `frontend/` to be created with own package.json

Workspace pattern:
```json
{
  "workspaces": ["cli", "frontend"]
}
```

This allows:
- Shared dependencies hoisted to root `node_modules/`
- Independent versioning per workspace
- Cross-workspace builds: `npm run build --workspaces`
- Isolation: frontend can use Vite while CLI uses tsc

### AO Registry Message Handlers

**Available Handlers:**
[Source: docs/prd/epic-6-developer-cli-frontend.md:167-174 - Message handlers]
[Source: architecture/data-models.md:86-115 - AO Registry Message Models]

Frontend will use (read-only via dryrun):

1. **Info** - Process metadata and capabilities
   - Action: "Info"
   - Returns: ADP v1.0 compliance info, handlers list

2. **Search-Skills** - Query-based search
   - Action: "Search-Skills"
   - Query: search string (empty = all skills)
   - Returns: Array of skill metadata

3. **List-Skills** - Paginated listing
   - Action: "List-Skills"
   - Limit: number of results
   - Offset: pagination offset
   - FilterTags: comma-separated tags
   - FilterName: name filter
   - Returns: Paginated skill array

4. **Get-Skill** - Specific skill retrieval
   - Action: "Get-Skill"
   - Name: skill name
   - Version: version (optional, latest if omitted)
   - Returns: Skill metadata with dependencies

5. **Get-Skill-Versions** - Version history
   - Action: "Get-Skill-Versions"
   - Name: skill name
   - Returns: Array of all versions with timestamps

**Frontend Usage Pattern:**
```typescript
// Type-safe query helper
async function searchSkills(query: string = ''): Promise<SkillMetadata[]> {
  const result = await dryrun({
    process: REGISTRY_PROCESS_ID,
    tags: [
      { name: 'Action', value: 'Search-Skills' },
      { name: 'Query', value: query }
    ]
  });

  // Parse response
  const data = JSON.parse(result.Messages[0].Data);
  return data.skills || [];
}
```
