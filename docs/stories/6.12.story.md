# Story 6.12: ux-ui-cursor-behavior-standards

## Status
Done

## Story

**As a** user of the Claude Agent Skills Registry frontend,
**I want** all clickable elements to display the pointer cursor and all non-clickable elements to show the default cursor,
**so that** I have clear visual feedback about which elements are interactive, improving the overall user experience and meeting production UX standards.

## Acceptance Criteria

1. All interactive button elements (`<button>`, shadcn/ui `<Button>`) display `cursor-pointer` on hover
2. All link elements (`<a>`, `<Link>`, `react-router-dom` navigation) display `cursor-pointer` on hover
3. All clickable badge components (category badges, tag badges) display `cursor-pointer` on hover
4. All clickable card components (SkillCard, FileCard with click handlers) display `cursor-pointer` on hover
5. All tab navigation elements (Tabs component in SkillDetail page) display `cursor-pointer` on hover
6. Non-interactive text, headings, and containers display default cursor (no pointer)
7. Disabled interactive elements display `cursor-not-allowed` state
8. Cursor behavior is visually verified using Playwright browser testing at desktop (1440px), tablet (768px), and mobile (375px) viewports
9. All cursor states are tested in the development environment using Playwright snapshot and screenshot tools
10. Changes follow existing Tailwind CSS utility patterns (e.g., `cursor-pointer`, `cursor-not-allowed`, `cursor-default`)
