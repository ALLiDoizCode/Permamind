### Architecture Context

**Tailwind CSS v4 Breaking Changes:**

Tailwind CSS v4 fundamentally changed theme configuration from JavaScript to CSS-based:

**v3 Syntax (OLD - doesn't work in v4):**
```javascript
// tailwind.config.js
export default {
  theme: {
    extend: {
      colors: {
        terminal: { bg: '#10151B' }  // ‚ùå IGNORED in v4!
      }
    }
  }
}
```

**v4 Syntax (NEW - required):**
```css
/* index.css */
@import "tailwindcss";

@theme {
  --color-terminal-bg: #10151B;  /* ‚úÖ Works in v4 */
}
```

**Current Issue:**

The frontend `tailwind.config.js` uses v3 syntax (lines 7-31 define colors in theme.extend), but Tailwind v4 ignores JavaScript config for colors. Result: `bg-terminal-bg` class compiles to NOTHING.

**Solution Options:**

**Option A: Migrate to v4 Syntax** (RECOMMENDED)
- Update `index.css` with `@theme` directive
- Remove color config from `tailwind.config.js`
- Keep v4 for modern features
- **Effort**: 30-60 minutes

**Option B: Downgrade to v3**
- Install `tailwindcss@^3.4.0`
- Keep existing `tailwind.config.js` (already correct for v3)
- Update `postcss.config.js` to use standard `tailwindcss` plugin
- **Effort**: 15-30 minutes
